## 🔍 Overview

<a href="#tampermonkey">Tampermonkey</a> serves Mac Safari users who want to customize, automate, or enhance website functionality beyond what the websites themselves provide and beyond what conventional browser extensions cover — power users, developers, and anyone who has found a userscript solving a specific browsing annoyance.

The <a href="#tampermonkey">power user customization use case</a> covers the enormous range of website annoyances and limitations that community userscripts address — removing intrusive elements, adding missing functionality to websites, automating repetitive web tasks, and modifying website appearance and behavior in ways the website's own developers never intended to support. The <a href="#tampermonkey">developer and scripter use case</a> uses Tampermonkey's built-in editor to write custom userscripts for personal automation needs — automating repetitive web-based tasks, extracting data from websites, or building personal productivity tools that run directly in the browser context.

The <a href="#tampermonkey">Greasy Fork and community script ecosystem</a> provides access to tens of thousands of community-written userscripts covering nearly any conceivable website modification — Tampermonkey is the runtime that makes this entire ecosystem accessible on Mac Safari. <a href="#tampermonkey">Safari-specific value</a> is particularly significant because Safari's extension ecosystem is smaller than Chrome's — Tampermonkey extends Safari's customization capability substantially by providing access to the userscript ecosystem that exists independently of any specific browser's extension store.</p>

<p align="center">
  <img src="https://raw.githubusercontent.com/wiki/OpenUserJS/OpenUserJS.org/images/tampermonkey_sa5.png" alt="Tampermonkey screenshot"/>

<a href="#tampermonkey">Privacy-respecting operation</a> runs userscripts locally within the browser without sending browsing data to external servers as part of Tampermonkey's own operation — userscripts themselves may make their own network requests as part of their functionality, but Tampermonkey's core operation respects user privacy. <a href="#tampermonkey">Active development and Safari compatibility</a> maintains Tampermonkey's function as Safari's extension APIs evolve, ensuring continued compatibility with current and future macOS Safari versions.

<a href="#tampermonkey">Script conflict management</a> handles situations where multiple installed userscripts might interact with the same website, providing tools to manage execution order and prevent conflicts between scripts. <a href="#tampermonkey">Free with optional support</a> — Tampermonkey is free to use with the full userscript management functionality, supporting continued development through optional contributions or premium features depending on current availability.</p>

## 💬 FAQ

<details>
<summary>What is the GM API in Tampermonkey?</summary>

The GM API is a Greasemonkey-compatible set of functions that give userscripts extended capabilities like cross-origin requests and persistent storage beyond standard webpage JavaScript.

</details>

<details>
<summary>Can Tampermonkey scripts conflict with each other?</summary>

Potentially, if multiple scripts modify the same website elements. Tampermonkey provides tools to manage execution order and reduce conflicts.

</details>

<details>
<summary>Does Tampermonkey work on all websites?</summary>

Userscripts are typically written for specific websites or URL patterns. Tampermonkey runs any installed script on the websites it's configured to target.

</details>

<details>
<summary>How do I disable a userscript temporarily?</summary>

Each installed script can be individually enabled or disabled from the Tampermonkey dashboard without uninstalling it.

</details>

<details>
<summary>Is Tampermonkey safe to use?</summary>

Tampermonkey itself is widely used and well-regarded. Individual userscripts vary in quality and trustworthiness, so reviewing scripts from reputable sources before installing is recommended.

</details>

<details>
<summary>Does Tampermonkey support keyboard shortcuts for scripts?</summary>

Some userscripts include their own keyboard shortcut functionality through the GM API, depending on how the individual script is written.

</details>
